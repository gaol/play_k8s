# Ansible Day-1 K8s Cluster Installation Design

## Overview

Ansible-based Day-1 automation that takes freshly provisioned VMs (from Day-0
`provisioner.py`) and produces a fully functional HA Kubernetes cluster with
CRI-O, Calico CNI, keepalived/HAProxy HA, and extensible operator installation.

## Config additions (config.yaml)

```yaml
k8s:
  container_runtime: "cri-o"
  cni: "Calico"
  k8s_version: "1.35"
  pod_network_cidr: "10.244.0.0/16"
  service_cidr: "10.96.0.0/12"
  operators:
    - lvm-operator

k8s_api_vip:
  ip: "192.168.150.50"
  hostname: "k8s-api"
```

## Directory structure

```
scripts/
  ansible/
    site.yml                        # main entry point
    inventory.ini                   # generated by provisioner.py
    group_vars/
      all.yml                       # generated — k8s versions, CIDRs, VIP, etc.
    roles/
      common/tasks/main.yml         # CRI-O, kubelet, kubeadm, kubectl, CNI plugins
      ha-control-plane/
        tasks/main.yml              # keepalived + HAProxy
        templates/
          keepalived.conf.j2
          haproxy.cfg.j2
      master-init/tasks/main.yml    # kubeadm init on first master
      master-join/tasks/main.yml    # kubeadm join --control-plane on master2,3
      worker/tasks/main.yml         # kubeadm join on workers
      calico/tasks/main.yml         # apply Calico manifests
      operators/tasks/main.yml      # loop over k8s.operators, run install.sh
    operators/
      lvm-operator/
        install.sh                  # install operator + apply storage config
        lvmcluster.yaml             # LVMCluster CR
        storageclass.yaml           # StorageClass definition
```

## Playbook execution order (site.yml)

| Order | Role               | Target        | Description                                      |
|-------|--------------------|---------------|--------------------------------------------------|
| 1     | common             | all nodes     | CRI-O, kubelet, kubeadm, kubectl, CNI plugins    |
| 2     | ha-control-plane   | masters       | keepalived + HAProxy for VIP failover             |
| 3     | master-init        | master1 only  | kubeadm init --control-plane-endpoint=k8s-api:6443 |
| 4     | calico             | master1 only  | apply Calico manifests                            |
| 5     | master-join        | master2,3     | kubeadm join --control-plane                      |
| 6     | worker             | workers       | kubeadm join                                      |
| 7     | operators          | master1 only  | copy and run install.sh for each configured operator |

## Role details

### common (all nodes)

1. Add CRI-O repo (`pkgs.k8s.io/addons:/cri-o:/stable:/v{k8s_version}`)
2. Add Kubernetes repo (`pkgs.k8s.io/core:/stable:/v{k8s_version}`)
3. Install packages: `cri-o`, `kubelet`, `kubeadm`, `kubectl`, `iproute-tc`
4. Enable CRI-O and kubelet services
5. Configure crictl to use CRI-O socket (`unix:///var/run/crio/crio.sock`)
6. Download and extract CNI plugins to `/opt/cni/bin`
7. Remove `zram-generator-defaults` (swap)

### ha-control-plane (masters)

1. Install `keepalived` and `haproxy` packages
2. Deploy `keepalived.conf` from template:
   - VRRP instance with VIP `192.168.150.50`
   - master1 = priority 101, master2 = 100, master3 = 99
   - Health check script monitors HAProxy
3. Deploy `haproxy.cfg` from template:
   - Frontend: bind VIP:6443
   - Backend: all 3 master IPs on port 6443 with health checks
4. Enable and start both services

### master-init (first master)

1. Run `kubeadm init` with:
   - `--control-plane-endpoint=k8s-api.k8s.local:6443`
   - `--pod-network-cidr=10.244.0.0/16`
   - `--service-cidr=10.96.0.0/12`
   - `--upload-certs`
   - `--apiserver-advertise-address=<master1_ip>`
2. Set up kubeconfig for the ssh user (`~/.kube/config`)
3. Capture join command + certificate key as Ansible facts

### calico (first master)

1. Apply Calico operator manifest (Tigera operator)
2. Apply Calico custom resource with `pod_network_cidr`
3. Wait for Calico pods to be ready

### master-join (master2, master3)

1. Run `kubeadm join` with `--control-plane` using token and cert key from master-init

### worker (workers)

1. Run `kubeadm join` using token from master-init

### operators (first master)

1. For each operator in `k8s.operators`:
   - Copy `scripts/ansible/operators/<name>/` directory to master1
   - Run `install.sh` (which may apply additional manifests like StorageClass)
2. Self-contained: each operator directory has its own scripts and manifests

## Network architecture

```
Client/kubectl
     |
k8s-api.k8s.local:6443 (VIP: 192.168.150.50)
     |
keepalived (VRRP) floats VIP across masters
     |
HAProxy (on whichever master holds VIP)
     |
     +--- master1:6443 (API server)
     +--- master2:6443 (API server)
     +--- master3:6443 (API server)
```

Pod network: `10.244.0.0/16` (Calico IPAM)
Service network: `10.96.0.0/12` (kube-proxy)
VM network: `192.168.150.0/24` (libvirt NAT)

## Provisioner integration

`provisioner.py generate` produces:
- `scripts/ansible/inventory.ini` — host groups from config.yaml nodes
- `scripts/ansible/group_vars/all.yml` — k8s settings from config.yaml

Playbooks and roles are static YAML files (not templated).

## Usage

```bash
# Day-0: provision VMs
python scripts/provisioner.py generate
cd scripts/run && ./virt-install-setup-network.sh && ./virt-install-create-vms.sh

# Day-1: install K8s
cd scripts/ansible
ansible-playbook -i inventory.ini site.yml
```
